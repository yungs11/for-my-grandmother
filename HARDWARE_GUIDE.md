# 🧸 할머니 말벗 에이전트 — 하드웨어 가이드 & 할 일 체크리스트

## 핵심 질문: 할머니가 가장 쉽게 쓸 수 있는 방법은?

할머니는 88세이시고, 기술에 익숙하지 않으시다. 따라서 **"할머니가 해야 할 행동이 적을수록 좋다"**가 원칙이다.

---

## 방법 비교

### 방법 A: 라즈베리파이 + 인형 (원래 계획)

**구성**: 라즈베리파이 + USB 마이크 + 스피커 + 터치 센서 → 인형 안에 내장

| 항목 | 내용 |
|------|------|
| 장점 | 완전 커스텀 가능, 인형이라 친근감, 터치만 하면 됨 |
| 단점 | 조립 필요, Wi-Fi 끊기면 먹통, 전원 항상 연결 |
| 예산 | 7~10만원 |
| 할머니 사용법 | 인형 꾹 누르기 → 대화 |
| 난이도 (개발) | ★★★☆☆ |

**예상 부품 (10만원 이하)**

| 부품 | 추천 제품 | 예상 가격 |
|------|-----------|-----------|
| 보드 | Raspberry Pi Zero 2W | ~2.5만원 |
| 마이크 | USB 미니 마이크 (쿠팡) | ~0.5만원 |
| 스피커 | 3W 미니 스피커 (3.5mm) | ~0.5만원 |
| 터치센서 | TTP223 정전식 터치 모듈 | ~0.2만원 |
| SD카드 | 32GB microSD | ~0.8만원 |
| 전원 | 5V 2.5A 어댑터 + 케이블 | ~1만원 |
| 인형 | 쿠팡/다이소 봉제인형 (30cm+) | ~1~2만원 |
| 기타 | 점퍼선, 납땜도구 | ~1만원 |
| **합계** | | **~7~9만원** |

> ⚠️ Pi Zero 2W는 Whisper small 모델이 느릴 수 있음. 예산 여유 시 Pi 4B(4GB) 추천 (+3만원)

---

### 방법 B: 안 쓰는 스마트폰 + 인형 ⭐ 추천

**구성**: 안 쓰는 안드로이드폰 → 간단한 앱/웹앱 설치 → 인형 옆이나 안에 배치

| 항목 | 내용 |
|------|------|
| 장점 | **마이크+스피커+Wi-Fi+배터리 다 내장**, 추가 구매 거의 없음 |
| 단점 | 폰 화면이 보일 수 있음 (인형 주머니에 넣으면 해결) |
| 예산 | **0~2만원** (인형값만) |
| 할머니 사용법 | 인형 꾹 누르기 → 대화 (폰 화면 큰 버튼 or NFC 태그) |
| 난이도 (개발) | ★★☆☆☆ |

**왜 이게 최고인가?**
- 라즈베리파이가 하는 일(마이크, 스피커, Wi-Fi, 터치)을 스마트폰이 이미 다 한다
- 배터리가 있어서 전원 코드 걱정 없음 (충전 독에 올려두면 됨)
- 음질이 라즈베리파이 + USB마이크보다 훨씬 좋음
- PWA(웹앱) 하나 만들면 앱 설치 필요 없음
- 현재 코드(Python)를 약간만 수정하면 웹 서버로 전환 가능

**필요한 것**
- 안 쓰는 안드로이드폰 (or 중고 2~3만원)
- 충전 케이블 + 어댑터
- 봉제인형 (주머니 달린 것)
- (선택) NFC 스티커 (~0.1만원) → 인형에 붙이면 폰 갖다 대기만 해도 앱 실행

---

### 방법 C: 태블릿 거치형

**구성**: 안 쓰는 태블릿 or 저가 태블릿 → 거치대에 세워두기

| 항목 | 내용 |
|------|------|
| 장점 | 화면이 커서 할머니가 보기 편함, 영상통화도 가능 |
| 단점 | 인형 느낌이 덜함, 할머니가 "기계" 느낌을 받을 수 있음 |
| 예산 | 0~5만원 (있는 태블릿 활용 시 0원) |
| 할머니 사용법 | 화면 큰 버튼 터치 → 대화 |
| 난이도 (개발) | ★★☆☆☆ |

---

### 방법 D: 클로바/구글홈 스마트 스피커 활용

| 항목 | 내용 |
|------|------|
| 장점 | 바로 사용 가능, 음질 좋음 |
| 단점 | **커스텀 불가** — 우리가 만든 메모리/프롬프트를 못 씀 |
| 예산 | 3~5만원 |

> ❌ 이 방법은 우리 프로젝트의 핵심(메모리, 맞춤 프롬프트, 보호자 알림)을 살릴 수 없어서 비추천

---

## 최종 추천: 방법 B (스마트폰) → 나중에 A (라즈베리파이)로 업그레이드

**이유:**
1. **지금 바로 시작 가능** — 집에 안 쓰는 폰 하나면 오늘이라도 테스트 가능
2. **돈이 안 듬** — 인형만 사면 됨
3. **할머니 반응을 먼저 확인** — 비싼 하드웨어 사기 전에 "할머니가 실제로 쓰시는지" 검증
4. **나중에 업그레이드** — 할머니가 잘 쓰시면 그때 라즈베리파이 + 인형으로 완성

---

## 윤수의 할 일 체크리스트

### Phase 0: 지금 바로 (오늘~이번 주)

- [ ] 집에 안 쓰는 안드로이드폰 찾기 (없으면 중고나라/당근에서 2~3만원짜리)
- [ ] `.env` 파일에 OpenRouter API 키 설정 확인
- [ ] `python3 main.py --text` 로 텍스트 대화 테스트 완료 ✅
- [ ] 마이크/스피커 테스트: 맥북에서 `python3 main.py` (음성 모드) 실행해보기
  - `pip3 install openai-whisper pyaudio` 필요
  - macOS에서 PyAudio: `brew install portaudio && pip3 install pyaudio`

### Phase 1: 스마트폰 프로토타입 (1~2주)

- [ ] 현재 코드를 **웹 서버 버전**으로 확장 (FastAPI + WebSocket)
- [ ] 스마트폰 브라우저에서 접속할 수 있는 웹 UI 만들기
  - 큰 버튼 하나: "할머니 여기 누르세요~"
  - 누르면 녹음 → 서버로 전송 → 응답 재생
- [ ] 스마트폰을 넣을 봉제인형 구매 (주머니 있는 것, 쿠팡/다이소)
- [ ] 할머니 댁 Wi-Fi 확인 (없으면 포켓 Wi-Fi or 폰 테더링)

### Phase 2: 할머니 테스트 (2~3주)

- [ ] 할머니 댁 방문하여 직접 시연
- [ ] 할머니 반응 관찰 (쓰시는지, 불편한 점, 음성 인식 잘 되는지)
- [ ] 피드백 수집하여 프롬프트/음성 설정 조정
- [ ] 텔레그램 보호자 알림 설정 (윤수 폰에 알림 오도록)

### Phase 3: 개선 & 안정화 (1개월)

- [ ] 할머니가 잘 쓰시면: 라즈베리파이 + 전용 인형으로 업그레이드 고려
- [ ] 선제적 대화 스케줄러 활성화
- [ ] 보호자 대시보드 웹 구축
- [ ] 오프라인 대비 (Wi-Fi 끊김 시 fallback 응답)

## 오디오 테스트 방법 (맥북에서)

### 1. 의존성 설치
```bash
# macOS에서 PyAudio를 위한 portaudio
brew install portaudio

# Python 패키지
pip3 install openai-whisper pyaudio
```

### 2. 마이크 테스트
```bash
# 마이크 목록 확인
python3 -c "import pyaudio; p = pyaudio.PyAudio(); [print(f'{i}: {p.get_device_info_by_index(i)[\"name\"]}') for i in range(p.get_device_count())]"
```

### 3. 음성 모드 실행
```bash
export OPENROUTER_API_KEY=sk-or-v1-xxx
python3 main.py --text --speak   # 텍스트 입력 + 음성 출력 (TTS 테스트)
python3 main.py                   # 풀 음성 모드 (STT + TTS)
```

> 💡 TTS가 Google Cloud 설정이 번거로우면, 먼저 `--text` 모드로 대화 품질을 확인하고
> 나중에 TTS를 붙이는 것도 좋은 방법!
